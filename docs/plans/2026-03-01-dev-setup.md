### Task 4: Initialize submodules and build GhosttyKit

**Step 1: Run setup.sh**

```bash
cd /Users/swanny/zed/crux && ./scripts/setup.sh
```

This script handles:
1. `git submodule update --init --recursive` (ghostty, bonsplit, homebrew-cmux)
2. Checks for zig in PATH
3. Builds GhosttyKit: `cd ghostty && zig build -Demit-xcframework=true -Doptimize=ReleaseFast`
4. Caches at `~/.cache/cmux/ghosttykit/<sha>/`
5. Symlinks `GhosttyKit.xcframework` at project root

Expected: Script completes without error. This step takes several minutes (Zig compiles Ghostty from source).

**Step 2: Verify submodules**

```bash
ls /Users/swanny/zed/crux/ghostty/build.zig && \
ls /Users/swanny/zed/crux/vendor/bonsplit/ && \
echo "Submodules OK"
```

**Step 3: Verify GhosttyKit**

```bash
ls -la /Users/swanny/zed/crux/GhosttyKit.xcframework
```

Expected: Symlink pointing to `~/.cache/cmux/ghosttykit/<sha>/`.

**Step 4: Verify Zig version compatibility**

```bash
grep minimum_zig_version /Users/swanny/zed/crux/ghostty/build.zig.zon
zig version
```

Expected: Installed version >= minimum. If setup.sh succeeded, this is confirmation.
