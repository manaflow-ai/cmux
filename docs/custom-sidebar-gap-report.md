# Custom Sidebar Interpreter — Gap Report

Generated by the `sidebar-idea-catalog` multi-agent workflow: 14 user personas each designed their ideal opinionated sidebar in the interpreter subset and reported missing features; deduped + prioritized below. Drives the interpreter self-improvement roadmap.

## Sidebar concept catalog

- **Backend/DevOps engineer** — Two-column ops cockpit: left local services with health dots/ports/restart-logs buttons + ports list + danger zone; right git/CI/deploy pipeline with one-tap rebuild/deploy and a deploy log tail. Color driven by status.
- **Frontend developer** — Triage dashboard: left per-workspace project health with tabs and server status; right 'what needs me' feed combining dev-server state, lint/typecheck/test results, and open-PR review status, each row jumping to the relevant terminal.
- **Data scientist (ML training)** — Training cockpit: left per-GPU util/VRAM/temp gauges + host RAM/CPU/disk; right active training runs (step/loss/ETA), dataset shortcuts that select workspace and run mount commands, and the live workspace list.
- **Technical writer** — Writing Desk: left manuscript files (tabs) with live word counts, budget bars, session total and a sprint timer; right publish lane with goal readout, format/lint, commit, and a gated publish button plus snippet actions.
- **Engineering manager (cmux)** — Manager triage board: left review queue grouped by CI-red / waiting-on-me / mergeable with tappable PRs and a merge button; right workspaces with tabs to refocus agent panes. Resizable.
- **Keyboard-first vim purist** — Minimal modeline HUD: top status strip (mode + leader + selected workspace), middle vim-style workspace/surface jump-list keyed with mnemonics, bottom two-column chord cheatsheet. Monospace, tight, one accent + one dim color.
- **SRE on call** — Command-center: left triage stack with paging banner (sev + ack), severity-sorted alerts with ack/runbook/logs taps, and runbook quick-launch; right live log tail of the selected service with a re-pipe-to-terminal action.
- **Open-source maintainer** — Maintainer triage cockpit: left workspace/PR navigator (workspace = PR under review) with quick-merge; right focused workspace's tabs (diff/terminal/test) with one-tap focus, plus sync and inbox-zero actions.
- **CS undergrad (pomodoro)** — Pomodoro command center: top session goal + live mm:ss countdown with hand-built progress bar; middle today's task checklist where each task launches its bound workspace; bottom 4-block pomodoro dot tracker and a day-streak flame.
- **Product designer (design system)** — Design cockpit: left workspace switcher + selected workspace's pipeline tabs (export/optimize/token build); right design system at a glance (brand palette swatches that copy hex, recent export thumbnails, live spacing/radius/type-scale token tables).
- **Power user (parallel agents)** — Agent-ops cockpit: left burn header + Approve/Deny inbox + agent roster with cost dots; right task queue grouped running/queued/done with promote/stop and cost-so-far. Color flags approval/idle/running/done.
- **Project manager (kanban)** — Three-lane kanban (To Do / Doing / Done) derived from workspace title prefixes, per-lane counts for WIP limits, due-date lines, an attention dot when tabs are focused, and one-tap workspace jump.
- **Multi-repo maintainer (notes)** — Working-memory sidebar: left always-open NOTES card for the selected workspace plus its tabs; right searchable, project-root-grouped workspace switcher with per-group counts, selected highlight, and one-tap focus.
- **iOS developer (build-run-debug)** — iOS build cockpit: left scheme switcher + grouped simulators/devices list with boot state and tap-to-boot/select; right big Build/Run/Test/Clean/Stop action pills, selected workspace tabs, and a recent-destinations strip.

### Detail

#### Richer live data context (PRs/CI, git branch+dirty, ports/processes, services/health, alerts/incidents, GPU/host telemetry, training runs, notes, tasks/agents cost, cwd files, clock, log/log-tail streams)  (14 personas, high)
- Rationale: Every single persona invented read-only context arrays beyond the documented workspaces/tabs. The interpreter only exposes workspaces/workspaceCount/selectedTitle, so almost every sidebar's primary content (alerts, PRs, services, GPUs, runs, notes, ports, git state) is fictional today. This is the universal blocker and the highest-leverage investment: most other gaps are cosmetic on top of data that does not exist.
- Example: `services = [{ id, name, port, up, healthy }]; git = { branch, dirty, ahead }; pulls = [{ number, ciState, needsMyReview }]; gpus = [{ index, utilPct, vramUsedGB, tempC }]; ports = [{ port, owner }]; clock.time`
- Effort: large — define a host-side provider protocol that populates named context objects (shelling out to git/lsof/gh/nvidia-smi/health probes) and inject them into the evaluation environment exactly like workspaces; needs a typed read-only value bridge and per-source refresh wiring.

#### Ternary operator (cond ? a : b)  (12 personas, high)
- Rationale: The single most-requested language primitive. Status-to-color/icon/label mapping is the visual core of every dashboard, and it must sit inside modifier arguments and string interpolation where if/else (a statement, not an expression) cannot go. Without it every one-line choice becomes a duplicated if/else view subtree, bloating a single-expression sidebar.
- Example: `.foregroundColor(build.status == "passing" ? "#34C759" : build.status == "running" ? "#FF9F0A" : "#FF453A")`
- Effort: small — add a ternary node to the expression grammar and evaluate the chosen branch; lowest-effort highest-impact change since the evaluator already has booleans and expression dispatch.

#### .background / .cornerRadius / shapes (RoundedRectangle, Rectangle, Capsule) / .fill / gradients  (12 personas, high)
- Rationale: Status pills, swatches, action-button fills, selected-row highlight chips, card chrome, and hand-built progress bars all need filled rounded backgrounds. Foreground-color-only rows read as an undifferentiated text wall, defeating the at-a-glance scannability that justifies a dashboard sidebar.
- Example: `Text(d.state).padding(4).background("#1C1C1E").cornerRadius(6)`
- Effort: medium — add shape view nodes plus .background(color-or-view), .cornerRadius, .fill; renderer must composite a background layer behind a view, which is more than the current flat text/stack rendering.

#### .frame(width/height/minWidth/maxWidth/alignment) and edge-specific padding  (11 personas, high)
- Rationale: Listed as known-missing. Needed for fixed/min column widths, equal-width action buttons, fixed-size thumbnails and progress cells, aligned monospace gutters, nested-row left indents, and title truncation/clipping. Without it rows size to content and wrap unpredictably, and authors can only push with Spacer.
- Example: `Text(pr.title).frame(maxWidth: 180, alignment: .leading); button.frame(maxWidth: .infinity, height: 56); .padding(.leading, 24)`
- Effort: medium — add a frame modifier carrying width/height/min/max/alignment plus edge-specific padding overloads; layout pass must honor explicit sizing constraints and ideal-size negotiation.

#### Auto-refresh / timers / .onAppear / live re-evaluation on new data  (11 personas, high)
- Rationale: Listed as known-missing. Every monitoring sidebar (services, CI, alerts, GPU gauges, run progress, pomodoro countdown, PR queue) is dead as a static snapshot. Sleep-based polling is banned, so the right model is host-driven: re-run the view expression whenever pushed context changes, plus an injected-clock-backed interval tick for countdowns.
- Example: `host re-renders on new data context; TimelineView(.periodic(by: 1)) for countdown; .onAppear { startPolling(every: 30) }`
- Effort: large — needs a reactive re-evaluation loop tied to context-change notifications and a testable injected Clock for bounded interval ticks; touches the runtime lifecycle, not just the parser.

#### Local mutable state (@State) + input controls (TextField, Toggle, Picker, Slider)  (9 personas, high)
- Rationale: Listed as known-missing. Blocks live search/filter switchers, label/hide-done/dark-variant toggles, sprint-length/word-goal/WIP pickers, editable commit messages, service-to-tail selection, and optimistic selection highlighting. Today every parameter is a baked literal and every 'input' is a dead affordance that only logs intent.
- Example: `@State var query = ""; TextField("filter", text: $query); Toggle("Hide done", isOn: $hideDone); Picker("label", selection: $label){...}`
- Effort: large — requires a state store with identity/persistence across re-renders, two-way bindings, and interactive control views with focus/keyboard handling; the biggest conceptual jump from a pure read-only renderer.

#### Array higher-order methods (.filter, .map, .reduce, .sorted, .flatMap, .first(where:), Set)  (9 personas, high)
- Rationale: Used for header rollups ('N/M up'), grouped/section counts (redCount/mergeableCount), cost/total aggregation, lane partitioning by predicate, severity sorting, flattening per-agent approval inboxes, deriving unique project roots, and finding the selected workspace. The subset only documents .count/.isEmpty, forcing hardcoded counts or host precomputation.
- Example: `services.filter { $0.up }.count; agents.reduce(0.0) { $0 + $1.costUSD }; workspaces.filter { $0.title.hasPrefix("[ ]") }; agents.sorted { $0.statusRank < $1.statusRank }`
- Effort: medium — implement higher-order methods over the interpreter's array values reusing the existing closure/`$0` machinery already present for ForEach; mostly a standard-library shim, no new evaluation model.

#### String methods (hasPrefix, contains, split, prefix/truncation, uppercased, lowercased, lastPathComponent, padding)  (9 personas, medium)
- Rationale: Listed as known-missing. Needed to bucket workspaces by title prefix (the kanban lane convention is impossible without hasPrefix), truncate long PR/workspace/log titles with an ellipsis, short-SHA, uppercase labels, derive a leaf folder name for group headers, and case-insensitive search matching.
- Example: `workspaces.filter { $0.title.hasPrefix("[ ]") }; Text(pr.title.count > 40 ? pr.title.prefix(40) + "…" : pr.title); root.lastPathComponent`
- Effort: medium — add a string method dispatch table; most are pure functions over the existing string value type, though path/locale-aware ones (lastPathComponent, localizedCaseInsensitiveContains) need host helpers.

#### Standalone ScrollView (with optional bottom anchor / pinned headers)  (9 personas, medium)
- Rationale: Listed as known-missing. HSplitView columns scroll, but authors repeatedly want independent scroll regions: a scrolling list above a pinned danger-zone/footer, a bottom-anchored log tail, a scrollable PR queue under a sticky search header, or scroll in single-column (non-split) layouts and the 3rd kanban lane.
- Example: `ScrollView { ForEach(deployLog) { line in ... } }.defaultScrollAnchor(.bottom)`
- Effort: medium — add a ScrollView node wrapping its child in a scroll container; modest renderer work plus optional scroll-anchor support, but no new evaluation semantics.

#### User-defined functions and reusable view helpers (func ... -> some View)  (8 personas, medium)
- Rationale: Listed as known-missing. Rows, badges, status derivations (health(ws), dueLine(task), isOverdue), and repeated card/button templates are inlined 2-4 times per sidebar, so a layout tweak must be copy-pasted everywhere and the single-expression body bloats. Reusable helpers are the main maintainability ceiling.
- Example: `func badge(_ text: String, _ color: String) -> some View { Text(text).padding(3).background(color).cornerRadius(4) }`
- Effort: large — requires named function definitions, a call stack with parameter binding/scoping, and view-returning closures evaluated lazily; expands the language from single-expression to multi-declaration.

#### Number and date/relative-time formatting  (8 personas, medium)
- Rationale: Listed as known-missing. Latency/durations, loss to 3 decimals, lr in scientific notation, currency cost ($1.23), percentages, zero-padded mm:ss, and relative timestamps ('3m ago', '2d ago') all currently rely on integer-division hacks or host-precomputed strings, pushing presentation into the data layer.
- Example: `cost.formatted(.currency(code: "USD")); Text(pr.openedAt, style: .relative); run.loss.formatted(.number.precision(.fractionLength(3)))`
- Effort: medium — expose a formatted() shim mapping to Foundation FormatStyle/RelativeDateTimeFormatter on the host; needs a Date value type in context plus a clock for relative formatting.

#### New action verbs: openURL, clipboard.write, file.reveal, surface.run, and multi-command per tap  (8 personas, high)
- Rationale: cmux(method,param:) and log() exist, but core interactions need verbs not in the documented surface: open a URL in browser (localhost server, PR page), copy a hex/token to the pasteboard, reveal an exported file in Finder, run a command in a surface (mount dataset, rebuild tokens, ship deploy), and sequence two cmux calls in one tap (focus + tail).
- Example: `openURL("http://localhost:3000"); cmux("clipboard.write", text: c.hex); cmux("surface.run", surface_id: id, command: "make mount"); Button{ cmux("surface.focus",...); cmux("sre.tail",...) }`
- Effort: medium — register additional host action handlers and allow a tap closure to contain multiple statements; the action dispatch and closure-body evaluation mostly exist, so this is wiring plus permission gating for destructive/shell verbs.

#### popover / sheet / menu / context menu / confirmationDialog  (6 personas, medium)
- Rationale: Listed as known-missing. Needed for transient notes editors anchored to a header glyph, right-click PR/agent/task secondary actions (approve, re-run CI, stop, copy URL, choose squash/rebase), label menus, and confirmation gates before destructive taps (Ship prod, Stop all, Kill port).
- Example: `row.contextMenu { Button("Re-run CI"){ cmux("ci.rerun", url: pr.url) } }; Button("Ship prod"){...}.confirmationDialog("Deploy to prod?", ...)`
- Effort: large — requires transient presentation state, anchor tracking, and dismissal lifecycle on top of the missing local-state model; menu items are themselves views with their own action closures.

#### Conditional/argument modifiers: .opacity, .lineLimit, .strikethrough, .bold(Bool), .disabled  (6 personas, medium)
- Rationale: Done tasks want strikethrough + dimmed opacity, long titles need lineLimit clamping, the active workspace wants conditional .bold(ws.selected), and a publish/merge button should be visibly .disabled until clean+at-goal rather than swapped for a different button via if/else.
- Example: `Text(t.title).strikethrough(t.done).opacity(t.done ? 0.55 : 1.0).lineLimit(1); Button("Publish"){...}.disabled(!clean)`
- Effort: small — add these as parameterized modifier nodes; each is a thin attribute on the rendered text/view, and .bold(Bool)/.disabled just take a boolean argument the evaluator already produces.

#### Explicit/monospaced fonts (.font(.system(size:weight:design:)), .monospaced(), .caption2)  (4 personas, medium)
- Rationale: The vim modeline and pomodoro timer depend on fixed-pitch glyphs for column alignment and non-jittering digits, and several sidebars want an explicit large monospaced size or a smaller .caption2 tier. The subset only lists preset semantic fonts with no size/design control.
- Example: `Text(remaining).font(.system(size: 40, weight: .bold, design: .monospaced)); Text(tab.title).monospaced()`
- Effort: small — extend the font modifier to accept system(size:weight:design:)/monospaced()/caption2, mapping directly to SwiftUI Font; purely a parameter-surface expansion.

#### Real image previews (AsyncImage / Image(file:) / Image(url:))  (2 personas, medium)
- Rationale: The designer's recent-exports strip is meaningless with SF Symbol placeholders; the point is seeing the actual rendered PNG/SVG/WEBP thumbnail the pipeline produced, loaded from disk or a URL.
- Example: `AsyncImage(url: URL(fileURLWithPath: "exports/hero@2x.png")) { $0.resizable().scaledToFill() }.frame(width: 36, height: 36)`
- Effort: medium — add file/url image nodes with async loading and a placeholder slot; requires sandboxed file-read access and async image decode, plus depends on .frame for sizing.

#### Optional chaining / nil-coalescing (?., ??)  (2 personas, low)
- Rationale: Listed as known-missing. Optional dictionary subscripts and 'selected workspace' lookups need safe defaults (notesByDir[dir] ?? [], a.runbookUrl ?? fallback); without ?? authors must guarantee every field exists, pushing nil-safety onto the host.
- Example: `let lines = notesByDir[w.directory] ?? []; cmux("link.open", url: a.runbookUrl ?? defaultRunbook)`
- Effort: small — add optional-value handling with ?. and ?? operators in the evaluator; dictionary subscripts already return a possibly-absent value, so this mainly formalizes nil semantics.

#### switch statement / expression  (2 personas, low)
- Rationale: Listed as known-missing. Multi-case state (sim Booted/Shutdown/Booting/Unavailable; task running/queued/done) collapses awkwardly into chained == comparisons and loses cases entirely. A switch would map each state to its icon+color cleanly.
- Example: `switch sim.state { case "booted": ...; case "booting": ...; default: ... }`
- Effort: medium — add switch parsing and case matching; if ternary/string-equality already exist it is mostly sugar, but supporting it as an expression that yields a view/value adds evaluation cases.

#### Tuple / struct destructuring and named fields in literals  (1 personas, low)
- Rationale: The vim cheatsheet encoded chords as [chord, desc] string pairs indexed by pair[0]/pair[1], which is fragile; a (key, desc) tuple or small struct with named fields would make literals self-documenting and let authors add columns without re-counting indices.
- Example: `for (key, desc) in [("gt", "next tab"), ("gT", "prev tab")] { ... }`
- Effort: medium — add tuple values and destructuring binding in for/let patterns; needs a new composite value type and pattern-binding support in the loop/let machinery.

#### .keyboardShortcut on buttons  (1 personas, low)
- Rationale: The parallel-agents cockpit wants Enter-to-approve on the inbox's primary action so approvals are keyboard-driven without a mouse round-trip.
- Example: `Button("Approve"){ cmux("agent.approve", request_id: req.id) }.keyboardShortcut(.return, modifiers: [])`
- Effort: medium — register a key equivalent for a button's action with the sidebar's responder/focus context; small parser surface but needs real keyboard-event routing into the interpreted view.

## Top priorities (implementation order)

1. Richer live data context — the universal blocker: 14/14 personas invent context beyond workspaces/tabs, so a host-side provider protocol injecting named read-only objects (git, ports, PRs/CI, services, telemetry, runs, notes, clock, log tails) unlocks the actual content of nearly every sidebar.
2. Ternary operator — smallest change with the highest reach (12/14): one expression-grammar node enables inline status-to-color/icon/label mapping everywhere modifiers and interpolation forbid if/else.
3. Array higher-order methods (filter/map/reduce/sorted/first(where:)) — medium effort reusing existing ForEach closure machinery; needed by 9 personas for rollups, counts, aggregation, lane partitioning, and sorting the context cannot pre-bake.
4. Auto-refresh / live re-evaluation on pushed data (+ injected-clock interval tick) — turns 11 static-snapshot monitoring sidebars into live ones via host-driven re-render, the SPEC-compliant alternative to banned sleep polling.
5. Visual layer: .background/.cornerRadius/shapes plus .frame sizing — together requested by ~12 and ~11 personas; jointly convert flat text rows into scannable pills/cards/aligned columns.
6. New action verbs (openURL, clipboard.write, file.reveal, surface.run) + multi-command taps — medium host wiring that makes 8 personas' core interactions do real work instead of only navigating or logging.
7. Local mutable @State + input controls (TextField/Toggle/Picker) — larger investment but the gate for every search/filter/toggle/picker affordance currently rendered as a dead stub (9 personas).
