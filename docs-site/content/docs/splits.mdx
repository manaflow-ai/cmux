# Split Panes

cmuxterm supports splitting panes within a workspace, allowing you to view multiple terminals side by side. See [Concepts](/concepts) for how panes fit into the hierarchy.

## Creating Splits

### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| **⌘D** | Split right |
| **⌘⇧D** | Split down |

### Using the CLI

```bash
# Split in a specific direction
cmuxterm new-split right
cmuxterm new-split down
cmuxterm new-split left
cmuxterm new-split up
```

## Navigating Splits

### Keyboard

Use **⌘⌥** + arrow keys to move between splits:

| Shortcut | Action |
|----------|--------|
| **⌘⌥←** | Focus left pane |
| **⌘⌥→** | Focus right pane |
| **⌘⌥↑** | Focus pane above |
| **⌘⌥↓** | Focus pane below |
| **⌘⇧L** | Flash focused panel |

### CLI

```bash
# List all surfaces in current workspace
cmuxterm list-surfaces

# Focus a specific surface
cmuxterm focus-surface <surface-id>
```

## Closing Splits

- **⌘W** - Close the focused surface (collapses the split if it's the last surface in a pane)
- If only one surface remains, **⌘W** closes the workspace

## Visual Styling

Unfocused panes are visually distinguished to help you identify the active pane.

### Configuration

In your Ghostty config:

```ini
# Dim unfocused panes (0.0 = fully dimmed, 1.0 = no dimming)
unfocused-split-opacity = 0.7

# Background fill for unfocused panes
unfocused-split-fill = #1e1e2e

# Color of the divider between panes
split-divider-color = #45475a
```

## Split Layout

cmuxterm uses a binary split tree structure:

- Each split creates two child panes
- Nested splits create a tree of panes
- Each surface has a unique ID accessible via `CMUX_SURFACE_ID`

### Example Layout

```
┌────────────────────────────────┐
│        Workspace 1             │
├──────────────┬─────────────────┤
│              │     Pane 2      │
│   Pane 1     ├─────────────────┤
│              │     Pane 3      │
└──────────────┴─────────────────┘
```

This layout is created by:
1. Split right → Pane 1 | Pane 2
2. Focus Pane 2
3. Split down → Pane 2 | Pane 3

## Sending Input to Specific Surfaces

With the CLI, you can send input to a specific surface:

```bash
# Send text to a specific surface
cmuxterm send-surface <surface-id> "echo hello"

# Send a keypress to a specific surface
cmuxterm send-key-surface <surface-id> enter
```
