- **Επιτάχυνση GPU** — Με τη βοήθεια του libghostty για ομαλή απόδοση

## Γιατί cmux;

Εκτελώ πολλές συνεδρίες Claude Code και Codex παράλληλα. Χρησιμοποιούσα το Ghostty με ένα σωρό διαιρεμένα τμήματα και βασιζόμουν στις εγγενείς ειδοποιήσεις του macOS για να ξέρω πότε ένας πράκτορας με χρειαζόταν. Αλλά το σώμα ειδοποίησης του Claude Code είναι πάντα απλώς «Ο Claude περιμένει την είσοδό σας» χωρίς συγκείμενο, και με αρκετές καρτέλες ανοιχτές δεν μπορούσα πλέον ούτε να διαβάσω τους τίτλους.

Δοκίμασα μερικούς ενορχηστρωτές προγραμματισμού αλλά οι περισσότεροι ήταν εφαρμογές Electron/Tauri και η επίδοση με ενοχλούσε. Επίσης απλώς προτιμώ το τερματικό αφού οι γραφικοί ενορχηστρωτές σε κλειδώνουν στη ροή εργασίας τους. Έτσι έφτιαξα το cmux ως εγγενή εφαρμογή macOS σε Swift/AppKit. Χρησιμοποιεί το libghostty για την απόδοση του τερματικού και διαβάζει την υπάρχουσα διαμόρφωση Ghostty για θέματα, γραμματοσειρές και χρώματα.

Οι κύριες προσθήκες είναι η πλαϊνή μπάρα και το σύστημα ειδοποιήσεων. Η πλαϊνή μπάρα έχει κάθετες καρτέλες που εμφανίζουν το git branch, την κατάσταση/αριθμό συνδεδεμένου PR, τον κατάλογο εργασίας, τις θύρες ακρόασης και το κείμενο της τελευταίας ειδοποίησης για κάθε χώρο εργασίας. Το σύστημα ειδοποιήσεων ανιχνεύει ακολουθίες τερματικού (OSC 9/99/777) και διαθέτει ένα CLI (`cmux notify`) που μπορείτε να συνδέσετε σε hooks πρακτόρων για Claude Code, OpenCode κ.λπ. Όταν ένας πράκτορας περιμένει, το τμήμα του αποκτά ένα μπλε δαχτυλίδι και η καρτέλα ανάβει στην πλαϊνή μπάρα, οπότε μπορώ να καταλάβω ποιος με χρειάζεται μεταξύ διαιρέσεων και καρτελών. Το Cmd+Shift+U μεταβαίνει στην πιο πρόσφατη μη αναγνωσμένη.

Το ενσωματωμένο πρόγραμμα περιήγησης διαθέτει ένα προγραμματιζόμενο API που μεταφέρθηκε από το [agent-browser](https://github.com/vercel-labs/agent-browser). Οι πράκτορες μπορούν να αποτυπώσουν το δέντρο προσβασιμότητας, να λάβουν αναφορές στοιχείων, να κάνουν κλικ, να συμπληρώσουν φόρμες και να αξιολογήσουν JS. Μπορείτε να διαιρέσετε ένα τμήμα προγράμματος περιήγησης δίπλα στο τερματικό σας και να έχετε το Claude Code να αλληλεπιδρά απευθείας με τον διακομιστή ανάπτυξής σας.

Όλα είναι προγραμματιζόμενα μέσω του CLI και του socket API — δημιουργία χώρων εργασίας/καρτελών, διαίρεση τμημάτων, αποστολή πλήκτρων, άνοιγμα URL στο πρόγραμμα περιήγησης.

## Το Ζεν του cmux

Το cmux δεν είναι κανονιστικό ως προς το πώς οι προγραμματιστές κρατούν τα εργαλεία τους. Είναι ένα τερματικό και πρόγραμμα περιήγησης με ένα CLI, και τα υπόλοιπα εξαρτώνται από εσάς.

Το cmux είναι ένα πρωτογενές στοιχείο, όχι μια λύση. Σας δίνει ένα τερματικό, ένα πρόγραμμα περιήγησης, ειδοποιήσεις, χώρους εργασίας, διαιρέσεις, καρτέλες και ένα CLI για να τα ελέγχετε όλα. Το cmux δεν σας επιβάλλει έναν προκαθορισμένο τρόπο χρήσης των πρακτόρων προγραμματισμού. Ό,τι φτιάχνετε με τα πρωτογενή στοιχεία είναι δικό σας.

Οι καλύτεροι προγραμματιστές πάντα έφτιαχναν τα δικά τους εργαλεία. Κανείς δεν έχει βρει ακόμη τον καλύτερο τρόπο να εργάζεται με πράκτορες, και σίγουρα ούτε οι ομάδες που φτιάχνουν κλειστά προϊόντα. Οι προγραμματιστές που βρίσκονται πιο κοντά στις δικές τους βάσεις κώδικα θα το ανακαλύψουν πρώτοι.

Δώστε σε ένα εκατομμύριο προγραμματιστές συνθέσιμα πρωτογενή στοιχεία και θα βρουν συλλογικά τις πιο αποδοτικές ροές εργασίας γρηγορότερα από όσο θα μπορούσε να σχεδιάσει εκ των άνω οποιαδήποτε ομάδα προϊόντος.

## Επαναφορά συνεδρίας

Η έξοδος από το cmux αποθηκεύει την τρέχουσα συνεδρία. Κατά την επανεκκίνηση, το cmux
επαναφέρει την κατάσταση που ανήκει στην εφαρμογή:
- Διάταξη παραθύρου/χώρου εργασίας/τμήματος
- Καταλόγους εργασίας
- Ιστορικό κύλισης τερματικού (κατά το δυνατόν)
- URL προγράμματος περιήγησης και ιστορικό πλοήγησης

Το cmux δεν αποθηκεύει σημείο ελέγχου για αυθαίρετη ζωντανή κατάσταση διεργασίας. Το tmux, το vim,
τα κελύφη και οι μη υποστηριζόμενες εφαρμογές τερματικού ανοίγουν ξανά ως κανονικά τερματικά.

Οι υποστηριζόμενες συνεδρίες πρακτόρων μπορούν να συνεχιστούν όταν τα hooks έχουν αποθηκεύσει ένα
εγγενές ID συνεδρίας. Εγκαταστήστε τα hooks αφού εγκαταστήσετε το CLI του πράκτορα ώστε το
εκτελέσιμό του να βρίσκεται στο `PATH`:

```bash
cmux hooks setup
cmux hooks setup codex
cmux hooks setup --agent opencode
```

Το `cmux hooks setup` εγκαθιστά τους υποστηριζόμενους πράκτορες που μπορεί να βρει και εκτυπώνει
μια σύνοψη για τους πράκτορες που παραλείφθηκαν. Οι υποστηριζόμενες ενσωματώσεις επαναφοράς
περιλαμβάνουν Claude Code, Codex, Grok, OpenCode, Pi, Amp, Cursor CLI, Gemini, Rovo Dev, Copilot,
CodeBuddy, Factory και Qoder. Το Claude Code χειρίζεται από το cmux Claude wrapper όταν η
ενσωμάτωση Claude είναι ενεργοποιημένη στις Ρυθμίσεις.

Οι προχωρημένοι χρήστες και οι ενσωματώσεις μπορούν να συνδέσουν μια προσαρμοσμένη εντολή
επαναφοράς στην τρέχουσα επιφάνεια τερματικού. Αυτό είναι χρήσιμο για εργαλεία με τη δική τους
ανθεκτική κατάσταση, όπως συνεδρίες tmux ή προσαρμοσμένα CLI πρακτόρων:

```bash
cmux surface resume set --kind tmux --checkpoint work --shell "tmux attach -t work"
cmux surface resume show --json
cmux surface resume clear --checkpoint work
```

Η σύνδεση παραμένει προσαρτημένη στην επιφάνεια cmux. Οι συνδέσεις που δημιουργούνται από το
δημόσιο CLI ή το socket αποθηκεύονται για επιθεώρηση και χειροκίνητη επαναφορά, εκτός αν εγκρίνετε
ένα υπογεγραμμένο πρόθεμα εντολής για αυτόματη επαναφορά. Τα εγκεκριμένα προθέματα συνδέονται
επίσης με τον κατάλογο εργασίας και τις ακριβείς τιμές περιβάλλοντος, όταν υπάρχουν. Ελέγξτε ή
επεξεργαστείτε τις εγκρίσεις στο **Ρυθμίσεις > Τερματικό > Εντολές επαναφοράς**. Το cmux εκτελεί
αυτόματα μόνο τις συνδέσεις επαναφοράς που σημειώνει ως αξιόπιστες, όπως συνδέσεις tmux που
ανιχνεύονται από ζωντανή διεργασία ή προθέματα εγκεκριμένα από τον χρήστη. Ευαίσθητα κλειδιά
περιβάλλοντος όπως tokens, κωδικοί πρόσβασης, μυστικά και κλειδιά API αφαιρούνται πριν αποθηκευτεί
μια σύνδεση επαναφοράς.

Για να διατηρήσετε τα επαναφερμένα τερματικά πρακτόρων αδρανή αντί να εκτελούν αυτόματα τις εντολές
επαναφοράς τους, απενεργοποιήστε το **Ρυθμίσεις > Τερματικό > Επαναφορά συνεδριών πρακτόρων κατά
το ξανά-άνοιγμα** ή ορίστε το εξής στο `~/.config/cmux/cmux.json`:

```json
{
  "terminal": {
    "autoResumeAgentSessions": false
  }
}
```

Αυτό απενεργοποιεί μόνο τις αυτόματες εντολές επαναφοράς πρακτόρων. Το cmux εξακολουθεί να
επαναφέρει την αποθηκευμένη διάταξη, τους καταλόγους εργασίας, το ιστορικό κύλισης και το ιστορικό
του προγράμματος περιήγησης.

Αν χρειάζεται να εφαρμόσετε ξανά χειροκίνητα το τελευταίο αποθηκευμένο στιγμιότυπο, χρησιμοποιήστε:
- `Αρχείο > Επαναφορά προηγούμενης συνεδρίας`
- `⌘ ⇧ O`
- `cmux restore-session`

Στο παρασκήνιο, το cmux γράφει ένα στιγμιότυπο με έκδοση στο
`~/Library/Application Support/cmux/` και τα hooks πρακτόρων γράφουν αντιστοιχίσεις συνεδριών στο
`~/.cmuxterm/`. Κατά την επαναφορά, το cmux αναδομεί πρώτα τη διάταξη, στη συνέχεια εκτελεί την
εγγενή εντολή επαναφοράς του υποστηριζόμενου πράκτορα όταν η αυτόματη επαναφορά πράκτορα είναι
ενεργοποιημένη.

Διαβάστε τον πλήρη οδηγό στο <https://cmux.com/docs/session-restore>.
