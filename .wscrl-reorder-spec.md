# Slice 3: reorder drag for the UITableView workspace list

## Context
Commit f30de01762 replaced the SwiftUI List with a UITableView (exact heights, zero
scroll-indicator churn). Reorder was left inert. This slice restores drag reorder with
EXACTLY the semantics the List's .onMove path had. All the semantic machinery already
exists and MUST be reused, not reimplemented:
- `WorkspaceListView.moveFlatRows(from:to:)` / `moveGroupedRows(from:to:)`
  (WorkspaceListView+DragDrop.swift) take SwiftUI onMove-convention indices
  (IndexSet source + pre-removal insertion destination) in the space of
  `displayedFlatWorkspaces` / `displayedGroupedListItems` (NO chrome row).
- `enablesWorkspaceReorder` gates the whole interaction (pipelining cap, filters,
  single-window, pinned rules).
- Per-row movability: workspace rows need `actionCapabilities.supportsMoveActions`;
  group headers move their whole group and need the ANCHOR workspace's
  `supportsMoveActions`; footers/chrome/filterEmpty never lift. This mirrors the old
  `.moveDisabled(...)` flags in WorkspaceListView.swift's List path.

## Implementation
1. Extend `WorkspaceListTable` inputs: `enablesReorder: Bool`,
   `moveRows: ((IndexSet, Int) -> Void)?` (nil when reorder unavailable), and enough
   per-item info for movability (workspacesByID/groupsByID already suffice).
   WorkspaceListView passes `enablesWorkspaceReorder` and a closure choosing
   `moveGroupedRows` vs `moveFlatRows` based on `rendersGroupedSections`.
2. Coordinator adopts `UITableViewDragDelegate` + `UITableViewDropDelegate`;
   `tableView.dragInteractionEnabled` tracks `enablesReorder`.
   - `itemsForBeginning`: return [] unless enabled && the item is movable (rules
     above). UIDragItem with `NSItemProvider()`, `localObject` = the table item.
   - `dropSessionDidUpdate`: only `session.localDragSession != nil` and single item →
     `UITableViewDropProposal(operation: .move, intent: .insertAtDestinationIndexPath)`;
     otherwise `.cancel`. Forbid destinations inside the chrome prefix (index < number
     of chrome items) with `.forbidden`.
   - `performDropWith`: map table rows to item-array indices by subtracting the chrome
     prefix count from both source and destination. Convert UIKit's destination to the
     SwiftUI onMove convention (`dest > src ? dest + 1 : dest` — VERIFY this against
     actual behavior by testing a downward and an upward drag; adjust if UIKit's
     coordinator semantics differ). Call `moveRows(IndexSet([src]), dest)`. Then
     `coordinator.drop(item.dragItem, toRowAt: destinationIndexPath)`.
   - The optimistic state mutation triggers updateUIView → snapshot apply; make sure
     the drop's own apply is not double-animated (apply with
     `animatingDifferences: false` when a drop just completed, mirroring how the
     coordinator already applies snapshots).
3. Fixture support so reorder is testable without a paired Mac: in
   WorkspaceListLayoutPreviewView (DEBUG), when
   `CMUX_UITEST_WORKSPACE_LIST_PREVIEW_REORDER=1`, hold the seeded workspaces in
   `@State` and pass a `moveWorkspace` closure that applies the intent to that local
   array (reuse `applyingWorkspaceMoveIntent` if convenient) and returns true. This
   both enables the gate (`moveWorkspace != nil`) and makes drops visibly persist.
